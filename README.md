# Site Visual Analyzer v1.1

Версия для быстрого визуального просмотра основных страниц WooCommerce в трёх разрешениях.

## Что исправлено

- корректно работает с WordPress, установленным в подпапке, например `/gelikon/`;
- `/`, `/cart/` и `/checkout/` теперь считаются путями внутри настроенного сайта;
- авторизация вынесена в отдельный файл `credentials.yaml`;
- поиск страниц выполняется уже после входа в аккаунт;
- анализатор открывает раздел заказов и находит ссылку на конкретный заказ;
- перед снимками корзины и checkout автоматически добавляется товар;
- если `product_url` пустой, выбирается первая найденная карточка товара.

## Первый запуск

### macOS / Linux

```bash
cd site-visual-analyzer
./install.sh
```

### Windows

Запустите `install.bat`.

## Авторизация

Откройте файл `credentials.yaml` и впишите данные тестового покупателя:

```yaml
auth:
  username: "test@example.com"
  password: "password"
```

Желательно использовать отдельного тестового пользователя, у которого уже есть хотя бы один заказ. Тогда анализатор сможет открыть:

- личный кабинет;
- список заказов;
- страницу конкретного заказа.

## Запуск

### macOS / Linux

```bash
./run.sh --clean
```

### Windows

```bat
run.bat --clean
```

После выполнения откроется:

```text
report/index.html
```

## Настроенный адрес

В `config.yaml` уже указан:

```yaml
site:
  base_url: "http://paveld9o.beget.tech/gelikon/"
```

Пути задаются относительно этой папки:

```yaml
url: "/cart/"
```

будет преобразован в:

```text
http://paveld9o.beget.tech/gelikon/cart/
```

## Корзина и оформление заказа

По умолчанию включено автоматическое добавление товара:

```yaml
prepare:
  add_product_to_cart:
    enabled: true
    product_url: ""
```

Пустой `product_url` означает: взять первый найденный товар.

Для стабильного результата лучше вставить адрес простого товара без обязательных вариаций:

```yaml
product_url: "/product/nazvanie-tovara/"
```

Для вариативного товара:

```yaml
variation_selectors:
  "select[name='attribute_pa_size']": "10"
```

## Важный момент по заказам

Авторизация сама по себе не создаёт заказ. Чтобы получить скриншот страницы отдельного заказа, у указанного пользователя должен существовать хотя бы один заказ, доступный в разделе «Мой аккаунт → Заказы».

---

## Оформление тестового заказа и страница «Заказ принят»

Версия 1.2 умеет автоматически пройти оформление заказа.

За один запуск создаётся **только один тестовый заказ**. После оформления анализатор сохраняет URL страницы вида:

```text
/checkout/order-received/123/?key=wc_order_...
```

И делает скриншоты этой страницы в Desktop, Tablet и Mobile.

Настройки находятся в `config.yaml`:

```yaml
order_flow:
  enabled: true
  payment_method: "cod"
```

`cod` — стандартный идентификатор оплаты при получении в WooCommerce. Возможные варианты зависят от сайта:

- `cod` — оплата при получении;
- `bacs` — банковский перевод;
- `cheque` — чек;
- идентификатор конкретного платёжного шлюза.

Тестовые ФИО, телефон, email и адрес лучше указывать в `credentials.yaml`:

```yaml
order_flow:
  billing:
    first_name: "Тест"
    last_name: "Покупатель"
    city: "Москва"
    address_1: "Тестовая улица, 1"
    postcode: "101000"
    phone: "+79990000000"
    email: "test@example.com"
```

После проверки тестовый заказ можно удалить из WooCommerce.

Чтобы временно не создавать заказ:

```bash
./run.sh --clean --skip-order
```

## Скриншот письма о заказе

Анализатор может подключиться к тестовому почтовому ящику по IMAP, найти письмо по номеру созданного заказа, сохранить его HTML и добавить в общий отчёт.

В `credentials.yaml`:

```yaml
email_capture:
  enabled: true
  imap_host: "imap.example.com"
  imap_port: 993
  username: "test@example.com"
  password: "пароль приложения"
  mailbox: "INBOX"
```

Email в `order_flow.billing.email` должен совпадать с ящиком, который проверяется через IMAP.

Для Gmail обычно нужен пароль приложения, а не основной пароль аккаунта. Для Яндекс Почты, Mail.ru и корпоративной почты также может потребоваться отдельный пароль приложения и включённый доступ по IMAP.

Если `email_capture.enabled: false`, заказ и страница «Заказ принят» всё равно будут проверены, но письмо в отчёт не попадёт.

## Медленная страница checkout

Страница оформления теперь открывается через мягкий режим ожидания. Анализатор не считает страницу сломанной только потому, что сторонние скрипты доставки, карты, аналитики или чата продолжают загружаться дольше 45 секунд.
